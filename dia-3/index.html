🎬 TalentoCine — Selección de butacas (en parejas, 60 min)

Construyan el selector de asientos de un cine aplicando reactive, computed, watch y v-model. El mapa de la sala y el diseño ya están armados: su trabajo es darle VIDA con la lógica reactiva. Al final lo publican en GitHub Pages y le pasan la URL al profe.

🅰️ Rol A — Estado & Butacas

Maneja la sala y lo que cuesta.

  • toggleAsiento(id) — elegir/soltar una butaca (las ocupadas no se tocan).
  • elegidos — computed con las butacas en estado "elegido".
  • cantidad — computed: cuántas butacas van elegidas.
  • subtotal — computed: cantidad × precio base.
  • total — computed: subtotal + recargo si la función es 3D.

🅱️ Rol B — Cliente & Reglas

Maneja al comprador y las validaciones.

  • puedeComprar — computed: hay butacas Y hay nombre.
  • watch sobre cantidad: avisar si eligen más de 6.
  • comprar() — si se puede, marca la compra como lista.
  • El v-model de cliente.nombre y del tipo de función.

🤝 Contrato compartido — acuérdenlo ANTES de codear

ESTADO (ya está dado, no lo borren):
  asientos          -> array reactive de { id, fila, col, estado }
                       estado puede ser: 'libre' | 'elegido' | 'ocupado'
  cliente.nombre    -> string  (v-model, Rol B)
  tipoFuncion       -> ref 'normal' | '3d'  (v-model, Rol B)
  PRECIO_BASE = 4500 · RECARGO_3D = 1500   (constantes dadas)

QUE COMPLETA CADA UNO:
  Rol A -> toggleAsiento(id) · elegidos · cantidad · subtotal · total
  Rol B -> puedeComprar · comprar() · watch(cantidad) · comprado (ref)

✅ Checklist — está listo cuando…

  • Al hacer clic, una butaca libre se pone verde (elegida) y vuelve a gris si la clicas de nuevo.
  • Las butacas ocupadas (gris oscuro) NO se pueden seleccionar.
  • Cada butaca elegida aparece como chip en "Tu compra" y suma al total.
  • Cambiar a función 3D recalcula el total (+$1.500 por butaca).
  • Si eligen más de 6 butacas, aparece el aviso amarillo (watch).
  • El botón "Comprar" solo se habilita con butacas + nombre, y muestra el mensaje de éxito.

Bonus: impedir seleccionar más de 6 en toggleAsiento · botón "vaciar selección" · descuento si compran 4 o más · mostrar la fila con más asientos libres.

🚀 Publicar en GitHub Pages y compartir la URL

Como es un solo index.html, NO hay que compilar nada. GitHub Pages lo sirve tal cual. Sigan estos pasos:

  1. Renombren el archivo a index.html (GitHub Pages busca ese nombre por defecto).
  2. Creen un repositorio en GitHub: github.com → New repository → nómbrenlo (ej: talentocine) → Public → Create.
  3. Suban el archivo: en el repo, Add file → Upload files → arrastren index.html → Commit changes.
    (O por consola: git init · git add . · git commit -m "cine" · git branch -M main · git remote add origin URL · git push -u origin main)
  4. Activen Pages: Settings → Pages → en Source elijan Deploy from a branch → Branch: main → carpeta / (root) → Save.
  5. Esperen ~1 minuto y recarguen. GitHub les mostrará la URL: https://TU-USUARIO.github.io/talentocine/
  6. Abran esa URL (¡pruébenla también en el celular!) para confirmar que funciona.
📨 Entrega: peguen la URL pública en el canal del curso / al profe, con los nombres de la pareja. Ejemplo: https://anita.github.io/talentocine/ — Ana & Beto
