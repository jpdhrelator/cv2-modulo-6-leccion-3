⚙ Preparación — lee esto antes de partir

Cada ejercicio se resuelve en su propio componente .vue con Composition API (<script setup>). Dentro de src/ de tu proyecto, organiza así:

src/
├── App.vue                // importa y muestra los ejercicios
├── style.css              // estilos de la práctica
└── components/
    ├── Ej1Semaforo.vue     // v-bind
    ├── Ej2Saludo.vue       // v-model básico
    ├── Ej3Modificadores.vue// .trim / .number / .lazy
    ├── Ej4Textarea.vue     // v-model en textarea
    ├── Ej5Checkbox.vue     // checkbox en 3 formas
    ├── Ej6Radio.vue        // radio
    ├── Ej7Select.vue       // select
    └── EjFinalReserva.vue  // desafío unificado

La estructura mínima que vas a repetir en cada componente es esta. Cópiala como punto de partida:

<script setup>
  import { ref } from 'vue'

  // TODO: declara tus variables reactivas con ref()
  // Con <script setup> NO hay return: todo lo que declaras acá
  // queda disponible en el template automáticamente.
</script>

<template>
  <!-- TODO: aquí va tu HTML con las directivas -->
</template>
Lo que cambia respecto a lo anterior: con <script setup> ya no existe el return. Todo lo que declaras en el bloque queda expuesto al template solo. Y ref, computed, etc. ya NO vienen de una variable global: los importas arriba con import { ref } from 'vue'.
En App.vue importa cada componente y móntalos (uno debajo del otro, o con botones para cambiar entre ellos — tú eliges).

1 · Semáforo con v-bind Básico

Tema: binding unidireccional · directiva v-bind / :

🎯 Objetivo: mostrar un "semáforo" (un círculo) que cambie de color según una variable, sin escribir el color a mano en el HTML.

📋 Instrucciones

  1. Crea una variable reactiva estado que arranque en "verde".
  2. Dibuja un <div> circular y píntale el color de fondo según el valor de estado.
  3. Agrega 3 botones ("Verde", "Amarillo", "Rojo") que cambien el valor de estado al hacer clic.
  4. Muestra debajo un texto: "Estado actual: {{ estado }}".

💡 Pistas

  • Para el color usa :style con un objeto: :style="{ background: ... }".
  • Para los botones piensa en el evento de clic (@click). Todavía no vimos eventos en profundidad, pero @click="estado = 'rojo'" te alcanza.
  • ¿Podrías cambiar también una clase con :class? Inténtalo si te sobra tiempo.

✅ Lo lograste si...

  • El círculo cambia de color al tocar cada botón.
  • El texto de abajo refleja el estado en vivo.
  • NO escribiste ningún color fijo dentro del atributo style del HTML.

2 · Saludo en vivo con v-model Básico

Tema: two-way binding · v-model en input

🎯 Objetivo: que al escribir tu nombre en un input, un saludo se actualice al instante en la pantalla.

📋 Instrucciones

  1. Crea una variable nombre que arranque vacía ("").
  2. Enlaza un <input> con esa variable en ambos sentidos.
  3. Muestra un <h2> que diga "¡Hola, [nombre]!" y se actualice mientras escribes.
  4. Si el input está vacío, que el saludo diga "¡Hola, visitante!" en su lugar.

💡 Pistas

  • El "ambos sentidos" te lo da UNA sola directiva. ¿Cuál era?
  • Para el caso del nombre vacío, dentro de las llaves {{ }} puedes usar el operador ||: {{ nombre || 'visitante' }}.

✅ Lo lograste si...

  • El saludo cambia letra por letra mientras escribes.
  • Al borrar todo, aparece "¡Hola, visitante!".

3 · Los modificadores importan Medio

Tema: .trim · .number · .lazy

🎯 Objetivo: demostrar con tus propios ojos por qué los modificadores de v-model no son un adorno.

📋 Instrucciones

  1. Crea DOS inputs de tipo número: a y b. A UNO ponle el modificador de número y al otro NO.
  2. Muestra el resultado de a + b para cada caso y observa la diferencia.
  3. Crea un input de texto usuario con el modificador que elimina espacios. Muestra su largo (.length) para comprobar que los espacios no se cuentan.
  4. Crea un input email con el modificador que actualiza al salir del campo. Fíjate que el valor NO cambia mientras escribes, solo al hacer clic afuera.

💡 Pistas

Sin el modificador de número, un input entrega texto. Y "2" + "3" en JavaScript da "23", no 5. Ese es justo el bug que este ejercicio te hace ver. Usa typeof para comprobar el tipo de dato.
Los tres modificadores se escriben pegados a v-model con un punto: v-model.___="...". Repasa la diapositiva de modificadores si no te acuerdas de los nombres.

✅ Lo lograste si...

  • Ves claramente que un caso SUMA (5) y el otro CONCATENA ("23").
  • Al escribir " hola " con espacios, el largo mostrado es 4, no 8.
  • El email solo se refleja al salir del campo.

4 · Bloc de notas con textarea Básico

Tema: v-model en textarea

🎯 Objetivo: un área de texto multilínea con contador de caracteres y vista previa que respete los saltos de línea.

📋 Instrucciones

  1. Crea una variable nota y enlázala a un <textarea>.
  2. Muestra cuántos caracteres lleva escritos.
  3. Muestra una vista previa del texto que conserve los saltos de línea.
  4. Extra: cambia el color del contador a rojo si supera los 200 caracteres.

💡 Pistas

  • Un <p> normal NO respeta los \n. ¿Qué etiqueta HTML sí lo hace? (empieza con "pre").
  • El largo de un string se obtiene con .length.
  • Para el color condicional, recuerda :style con un ternario.

✅ Lo lograste si...

  • Escribes varias líneas y la vista previa las muestra separadas.
  • El contador sube en tiempo real.

5 · Checkbox en sus 3 formas Medio

Tema: checkbox booleano · array · :true-value/:false-value

🎯 Objetivo: dominar los tres comportamientos del checkbox con v-model.

📋 Instrucciones

  1. Booleano: un checkbox "Suscribirme al newsletter" enlazado a una variable que valga true/false. Muéstrala.
  2. Array: tres o más checkboxes de "hobbies" (leer, deporte, música...) que compartan el mismo v-model. Muestra el array resultante y cuántos hay marcados.
  3. Valores personalizados: un checkbox "Modo" que guarde "pro" cuando está marcado y "gratis" cuando no. (Nada de true/false aquí).

💡 Pistas

  • Para el caso array, la variable debe arrancar como ref([]) y cada checkbox necesita su atributo value.
  • Para los valores personalizados busca los atributos :true-value y :false-value. Ojo con las comillas: :true-value="'pro'".

✅ Lo lograste si...

  • El booleano muestra true/false.
  • Al marcar hobbies, el array crece y decrece.
  • El "Modo" muestra "pro" o "gratis", nunca true/false.

6 · Encuesta con radio Básico

Tema: radio con v-model

🎯 Objetivo: una pregunta con opciones donde solo se puede elegir UNA.

📋 Instrucciones

  1. Pregunta: "¿Cómo calificarías la clase?" con opciones: Excelente, Buena, Regular, Mala.
  2. Todas las opciones deben compartir el mismo v-model.
  3. Muestra abajo la opción elegida.
  4. Extra: muestra un mensaje distinto según la respuesta (por ejemplo, "¡Gracias!" si es Excelente o Buena).

💡 Pistas

  • La regla de oro del radio: mismo v-model para todo el grupo, y cada uno con su value.
  • La variable guardará automáticamente el value del seleccionado. No tienes que recorrer nada.
  • Para el mensaje extra, un v-if / v-else te sirve.

✅ Lo lograste si...

  • Solo se puede marcar una opción a la vez.
  • El texto refleja la elección al instante.

7 · Elige tu plan con select Medio

Tema: select con v-model

🎯 Objetivo: una lista desplegable que enlaza el plan elegido y muestra su precio.

📋 Instrucciones

  1. Crea un <select> con planes: Básico, Pro, Premium.
  2. Agrega una primera opción deshabilitada como "Selecciona un plan" (placeholder).
  3. Muestra el plan elegido y su precio (invéntalos: por ej. Básico $0, Pro $9.990, Premium $19.990).
  4. Extra: deshabilita un botón "Contratar" hasta que se elija un plan.

💡 Pistas

  • El placeholder es una <option disabled value=""> y la variable arranca en "".
  • Para el precio, puedes guardar un objeto { basico: 0, pro: 9990, ... } y leerlo según el plan elegido.
  • Para deshabilitar el botón: :disabled. Ya lo usaste en el ejercicio 1, ¿te acuerdas?

✅ Lo lograste si...

  • El precio cambia según el plan.
  • El botón "Contratar" está bloqueado hasta elegir.

🏆 Ejercicio unificado — Reserva de hotel Desafío

Tema: TODO junto · input · modificadores · textarea · checkbox · radio · select · v-bind

🎯 Objetivo: construir UN formulario completo de "Reserva de hotel" que use todos los tipos de campo vistos en la lección y muestre un resumen en vivo de la reserva. Este es el que demuestra que de verdad entendiste la clase.

📋 Requisitos — tu formulario debe incluir

  1. Nombre del huésped → input de texto sin espacios sobrantes (modificador correspondiente).
  2. Cantidad de noches → input numérico que sume de verdad (modificador correspondiente).
  3. Tipo de habitación → un select (Individual, Doble, Suite) con precios por noche distintos.
  4. Régimen → radio (Solo alojamiento, Desayuno incluido, Pensión completa).
  5. Servicios extra → checkbox en modo array (WiFi premium, Estacionamiento, Spa...), cada uno con su precio.
  6. ¿Fumador? → un checkbox con valores personalizados que guarde "fumador" / "no-fumador".
  7. Peticiones especiales → un textarea.
  8. Acepto la política de cancelación → checkbox booleano obligatorio.
  9. Resumen en vivo → un panel que muestre TODOS los datos elegidos y el precio total calculado (noches × precio habitación + extras).
  10. Botón "Reservar" → deshabilitado (:disabled) mientras falten el nombre, la habitación o la aceptación de la política.

💡 Pistas

  • Arranca declarando TODAS las variables reactivas en el setup(). Una por cada campo.
  • Para el precio total, la lógica es: precioHabitacion * noches + suma de extras. Piensa cómo sumar los extras del array (cada extra tiene un precio).
  • Para el botón: la condición de "deshabilitado" es una expresión que combina varias comprobaciones con && y ||.
  • El resumen en vivo son solo {{ }} leyendo tus variables. Si el binding está bien, el resumen se actualiza SOLO.
  • Formatea el precio bonito con .toLocaleString('es-CL').
Ojo, no te saltes esto: si el precio total te da NaN o algo raro, casi seguro es porque un input te está entregando texto en vez de número. Ya sabes qué modificador arregla eso. Ese error es INTENCIONAL en el diseño del ejercicio: quiero que lo encuentres tú.

✅ Lo lograste si...

  • Usaste los 7 tipos de campo de la lección al menos una vez.
  • El precio total se recalcula solo al cambiar cualquier campo.
  • El botón "Reservar" se bloquea/desbloquea según los campos obligatorios.
  • El resumen refleja exactamente lo que elegiste, sin tocar el DOM a mano.
  • No hay ningún NaN en el precio.
